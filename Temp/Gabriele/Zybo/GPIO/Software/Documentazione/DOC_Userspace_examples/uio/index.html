Applicativo GPIO driver UIO
GPIO con driver UIO

Introduzione

GPIO uio è un esempio di utilizzo, a livello user space, del driver uio per interfacciarsi con la periferica custom GPIO sintetizzata sulla PL della board ZYBO.
Tale esercizio ha lo scopo di interfacciarsi a livello user space del SO Linux con la periferica GPIO tramite il modulo kernel generico UIO.

Use

Step 1: Configurare il CrossCompile

Modificare il file init_crosscompile_build_environment.sh cambiaando la variabile BUILDROOT_PATH con il path d'installazione di buildroot, oppure mettere il path d'installazione del proprio crosscompilatore facendo però attenzione a modificare anche la variabile CROSS_COMPILE assegnandovi il nome del crosscompilatore che si intende utilizzare.

Step 2: Esportare le variabili d'ambiente

Aprire il terminale, posizionarsi nella directory corretto e digitare il seguente comando:

source init_crosscompile_build_environment

Step 3: Cross Compilazione

Usando lo stesso terminale in cui si è digitato il comando source, posizionarsi nella cartella dei sorgenti e digitare il seguente comando:

make

Step 4: Trasferimento dell'eseguibile sulla board ed esecuzione

Assicurarsi che il device tree sia quello corretto e che dia le giuste informazioni al kernel linux affinchè sia possibile associare un mdoulo generico, UIO, alla periferica.
Copiare il file main_no_int o main_int sulla microsd contenente il root FS. La cartella in cui si copia è indifferente, basta solo tenerla presente.
Avviare la zybo. Connettersi utilizzando la seriale (per i parametri vedere il manuale).
Posizionarsi nella cartella contenete l'eseguibile copiato precedentemente e dare il comando

main_no_int -w hex_value

per l'eseguibile main_no_int
e dare il comando

main_int

per l'eseguibile main_int
